## Overview

This system provides a comprehensive solution for managing patient records in a hospital setting. It uses a Binary Search Tree to store and organize patient data, enabling fast search, insertion, deletion, and retrieval operations with O(log n) time complexity.

## Features

- ✅ **Add New Patients**: Insert patient records with unique IDs
- ✅ **Search by ID**: Fast O(log n) search using patient ID
- ✅ **Search by Name**: Search patients by name (case-insensitive, partial match)
- ✅ **Update Records**: Modify existing patient information
- ✅ **Delete Patients**: Remove patient records by ID
- ✅ **Display All**: View all patient records sorted by ID
- ✅ **Test Data**: Pre-loaded sample patient records for testing

## Project Structure

The project consists of 4 Java classes:

1. **Patient.java** - Data class representing a patient with:

   - Patient ID (integer)
   - Name (String)
   - Age (integer)
   - Gender (String)
   - Medical History (String)

2. **Node.java** - BST node structure containing:

   - Patient object
   - Left child pointer
   - Right child pointer

3. **PatientBST.java** - Binary Search Tree implementation with:

   - Insert operation
   - Search by ID
   - Search by Name
   - Delete operation
   - Display all (in-order traversal)

4. **HospitalSystem.java** - Main application with:
   - Interactive menu system
   - User input handling
   - Test data loader

## Usage Guide

### Main Menu Options

1. **Add New Patient**

   - Enter patient ID (must be unique)
   - Enter patient name
   - Enter age (must be a number)
   - Enter gender
   - Enter medical history

2. **Search Patient by ID**

   - Enter the patient ID to search
   - System displays full patient record if found

3. **Search Patient by Name**

   - Enter full or partial name
   - System displays all matching patients (case-insensitive)

4. **Update Patient Information**

   - Enter patient ID to update
   - Press Enter to keep current value, or type new value
   - Can update: Name, Age, Medical History

5. **Delete Patient**

   - Enter patient ID to delete
   - System confirms deletion

6. **Display All Patients**

   - Shows all patient records sorted by ID (in-order traversal)

7. **Exit**
   - Closes the application

## Data Structure

**Binary Search Tree (BST)**

- **Ordering**: Patients are organized by Patient ID
- **Time Complexity**:
  - Insert: O(log n) average case
  - Search by ID: O(log n) average case
  - Search by Name: O(n) - requires full tree traversal
  - Delete: O(log n) average case
  - Display All: O(n) - in-order traversal

## Error Handling

The system handles various error cases:

- Duplicate patient IDs
- Invalid number formats (for ID and Age)
- Patient not found (for search, update, delete)
- Empty input validation

## Notes

- Patient IDs must be unique integers
- The system is case-insensitive for name searches
- All data is stored in memory (not persisted to disk)
- The BST maintains sorted order by patient ID automatically
